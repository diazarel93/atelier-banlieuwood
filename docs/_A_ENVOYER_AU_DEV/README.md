# BANLIEUWOOD — Dossier développeur

Installe ce dossier dans ton espace de travail. **Sa structure ne change pas d'un cycle à l'autre.**

---

## 3 fichiers à connaître

| Fichier | Ce que c'est |
|---|---|
| `FEUILLE_ROUTE_ACTIVE.md` | Ce qu'on te demande pour le cycle en cours. **Lire en premier.** |
| `rapport/RAPPORT_EN_COURS.md` | Ton rapport d'état à remplir et nous renvoyer. |
| `rapport/TEMPLATE_RAPPORT.md` | Modèle vierge si tu veux repartir de zéro. |

Le dossier `archives/` contient tes rapports précédents — classés automatiquement de notre côté.

---

## Le cycle en 3 étapes

```
1. TU REÇOIS   →  une FEUILLE_ROUTE_ACTIVE.md mise à jour  →  tu lis, tu travailles
2. TU ENVOIES  →  rapport/RAPPORT_EN_COURS.md rempli        →  tu nous le renvoies
3. NOUS GÉRONS →  on archive l'ancien, on met à jour la feuille de route, on repart
```

C'est tout. Même structure, mêmes fichiers, cycle après cycle.

---

## Règles simples

- **Ne pas renommer les fichiers.** Les noms sont stables pour faciliter le traitement automatique.
- **Toujours remplir tout le template** — même si c'est "RAS". Les sections vides nous informent.
- **Dire la vérité sur l'état réel.** Un rapport flou produit une feuille de route imprécise — et plus de travail pour toi ensuite.
- **Signaler les décisions attendues.** Si tu attends une décision de notre côté avant d'avancer, note-le dans la section "Décisions attendues" — ne pas attendre d'être bloqué.
